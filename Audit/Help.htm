1.     Overview

Cloud ECM Audit is product that allows user monitor the actions occur during time system operates. This also allows user monitor the history of document in system.

Audit product consists of features:

         Action log: This feature allows user monitor the action that user operates in system.

         Report management: This feature allows user show the report. Reports will be built by customer requirement.

         Document History: This feature allows you monitor the versioning of document. The document history consists of two kinds:

o   History of existing document.

o   History of deleted document.

2.     Action Log

This feature allows user monitor the actions that user operates on system.

Figure 2.1 Action log view

         Login to system with appropriate account.

         Click on Action Log button. Action log view will display.

         User click on Next button to go to next page.

         User click on Previous button to back to previous page.

a.      Search Action Log

         This feature allows user search the action log by condition.

         In Action Log view, select expand button in top-right of Action Log screen.

         Enter search conditions to search panel. The search conditions consist of:

         User name: This condition allows user filter action log by User name with specific condition.

         Logged date: This condition allows user filter action log by logged date.

         Action name: This condition allows user filter action log by action name.

         User click on Search button to search action log. The result will show on action log list.

Figure 2.2 Search action log view

b.      Delete Action Log

This feature allows user delete the action logs.

         User select action log(s) need to delete.

         User click on Delete button to delete selected action log(s).

3.     Report

Report feature will be built by customer requirement. The user guide will be updated later.

4.     Document History

This feature allows user monitor the change of documents. The history consists of existing documents and deleted documents.

a.      Document Version

This feature allows user monitor the version of documents that exist in system.

         Login to system with appropriate account.

         Click on History drop-down button. Menu item will display.

         Select on Document Version menu item. Document Version screen will display.

Figure 4.1 Document history menu item

This feature allows user search document to monitor version.

Content Type search

This search is similar to Content Type search on Archive product. Please refer Archive User Guide.

    II.            Document Version

This feature allows user monitor all change in each document that system users operate on system.

         In search result document list, select document need to monitor versioning.

         Double-click on select document item or right-click on item and select View History menu item. Document version screen will display.

Figure 4.5 Document versioning view

Document View

Please see Document View in Archive User Guide.

Dashboard Tab

This tab presents information about document.

Figure 4.6 Dashboard tab view

Document Information: This panel contains the information about current version of document.

Versions: This panel contains the information about document versioning. The panel shows the list of document version of document.

         User double-click on version item in version list. The detail of document versioning will display.

         User click on Show current content to display current version of document.

Activities tab

This tab presents the information about the activities that user operated on document.

Figure 4.7 Activities tab view

Information of activities consists of:

         User action on document.

         Action message on document.

         IP address of machine action on document.

         Action name.

         Action logged date.
