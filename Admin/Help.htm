1.     Overview

 

Cloud ECM – Admin is administration tool to help admin user to configure and setup data to run Cloud ECM – Archive and Cloud ECM – Audit systems. This product consists of:

·         User Group Management: manage User Groups.

·         User Management: manage User information.

·         Content Type Management:  create the Content Type and setup configurations for the Content Type.

·         Permission Management: configure the Permission of User Group against the Content Type

·         Settings: Other configurations of the system.
 

2.     Login

 

Figure 2.1 Login screen

User must provide user name and password to access admin module, this admin module is for admin user only, system denies non-admin logging into admin module.

Note: an admin user was created when we run DBGenerator, when user first logs into the system, he/she will use this account to login and create another accounts

Login to system

·         Enter User Name and Password.

·         Click on “Sign in” button.

·         System navigates to main screen of the admin module if user name/password are correct and this user is admin user, if not then there is an error message get displayed on the screen

Define Content Type Field

Field is the property of the Content Type; you can define as many fields as you want to manage the Content Type. Value of the Field will be entered during the creation of the Content Type

·         Click “Add” button to open “Field properties” screen.

 

Figure 3.4 Field properties view

·         Fill all required and valid data.

o   Field name: field name, this text value can contains maximum 255 characters. The name is unique on Content Type.

o   Data type: data type of the field, it can be

§  String

§  Integer

§  Decimal

§  Datetime

§  Boolean: This is logic type.

§  Picklist: This type is defined by user.

o   Is required: this field is mandatory or not

o   Is restricted: this field is available to view to all users or just users that have the permission to view it (permission “View restricted field” in the Permission module)

o   Max length: maximum characters that user can enter; apply to String data type only.

o   Default value: This property defines the default value of field when the Content Type is created.

4.     Permission Management.

 

This feature allows user to assign permission for User Group on Content Type. There are three permission levels exist in the system.

·         Content Type Permission.

·         Annotation Permission.

·         Audit permission.

a.       View permissions.

 

Click “Permission” menu item in left hand side menu. Permission view will display with two display option.

o   View by User Group

o   View by Content Type

Figure 4.1 Permission view by Content Type

Figure 4.2 Permission view by User Group

b.       Assign permissions

 

This feature allows user assign permission to User Group for Content Type.

·         Click on User Group or Content Type in permission list to display permission detail view.

·         Check on permission item to assign permission to User Group for Content Type. The permissions are classified to three types:

o   Document permission: Those permissions apply for document level.

§  Capture: Check this option to allow user access Capture function to capture document.

§  Search: Check this option to allow user access Search function to search document in system.

§  Update field value: Check this option to allow user update Field value.

§  View restricted field: Check this option to allow user see restricted field.

§  Append page: Check this option to allow user append one or more page to existing pages of document.

§  Replace page: Check this option to allow user replace one or more pages by another pages.

§  Rotate page:  Check this option to allow user rotate existed or captured pages.

§  Delete page: Check this option to allow user delete existed or captured pages.

§  Hide all annotation: Check this option to allow user hide all existed annotation.

§  Export field values: Check this option to allow user export field values to file system.

§  Send mail: Check this option to allow user send document via email.

§  Print: Check this option to allow user print document.

o   Annotation permissions:

§  Redaction: Check this option to allow user select all redaction permission.

·         Add: Check this option to allow user add redaction to pages.

·         Delete:  Check this option to allow user delete redaction.

§  Highlight: Check this option to allow user select all highlight option.

·         Add: Check this option to allow user add highlight to pages.

·         Delete: Check this option to allow user delete highlight.

·         View: Check this option to allow user view highlight.

§  Text: Check this option to allow user select all text option.

·         Add: Check this option to allow user add comment to pages.

·         Delete: Check this option to allow user delete comment.

·         View: Check this option to allow user view comment.

o   Audit permission:

§  Audit: Check this option to allow user view document history and versioning.

§  View log: Check this option to allow user view action log.

§  Delete log: Check this option to allow user delete action log.

§  View report: Check this option to allow user view report.

§  Restore document: Check this option to allow user restore document.

7.     Settings.

This feature allows user setup the configuration data for system.

·         Login to system with correct account.

·         Select “Settings” menu item in left hand side menu. The setting view will display.

·         Number of rows per retrieved search result: This field defines the number of rows will be returned when user run search function.

·         OCR auto correct configuration: This feature allows user define the set of pair works or characters wrong/right often occur when OCR function running. Base on set of those pair system will repair the incorrect word to correct automatically.

Figure 7.1 Settings view

To define the OCR Auto Correct data, user follows by below steps:

·         In Settings view, user clicks on “Configure” button. The OCR Auto Correct screen will display.

·         Select language to view the definition for each language. Each language has maximum two set of auto correct definition, one for Unicode and one for none Unicode.

·         Click on “Close” button to exit to OCR Auto Correct definition screen and back to Settings view.

Figure 7.2 OCR Auto correct definition view

Figure 7.3 Auto correct definition view with selected language

a.       Add new OCR auto correct definition

·         Figure 7.4 Auto correct definition view with selected language

·         Click on “Add” button to open empty pane in right hand side to define auto correction data.

·         Enter data by pair following format:

o   [Incorrect word] = [Correct word]

o   Each pair incorrect/correct word separated by enter key on keyboard.

·         Support Unicode: Check this option to determine this definition use for Unicode format.

·         Click “Save” button to save data.

·         Click on “Cancel” button to cancel add new action.

b.       Update OCR auto correct definition

This feature allows user update existing OCR auto correct definition.

·         In OCR Auto Correct Definition view, select language to display OCR auto correct list.

·         Select definition item need to update. This definition will be shown on right hand side pane to edit and update.

·         Update data.

·         Click on “Save” button to save change.

·         Click on “Cancel” button to cancel update action.

Figure 7.5 Update OCR Auto Correct definition view

c.        Delete OCR Auto Correct definition

This feature allows user delete OCR Auto Correct definition.

·         In OCR Auto Correct Definition view, select item needs to delete.

·          OCR Auto Correct definition will be removed.
